## Real-World Use Cases
1. **Kubernetes Liveness and Readiness Probes**: You deploy a Spring Boot microservice to Kubernetes. Kubernetes needs to know if the application is healthy enough to receive web traffic, or if it has crashed and needs to be restarted. By simply adding the `spring-boot-starter-actuator` dependency, Spring Boot exposes a `/actuator/health` endpoint. You configure Kubernetes to ping this URL every 10 seconds. If Actuator returns an HTTP 200, Kubernetes keeps routing traffic to it. If it returns HTTP 503, Kubernetes stops sending traffic and restarts the pod.
2. **Prometheus Metrics Scraping and Grafana Dashboards**: Your operations team needs to monitor the latency of the `/api/checkout` endpoint and the memory usage of the JVM. You expose the `/actuator/prometheus` endpoint. A Prometheus server continuously scrapes this endpoint, pulling thousands of metrics automatically generated by Micrometer. Grafana connects to Prometheus and displays these metrics on a beautiful real-time dashboard, allowing the team to set up automated alerts if memory usage exceeds 85%.

## Beginner Level Questions

### Q1. What is Spring Boot Actuator? [Easy]
**Answer:** Actuator is a sub-project of Spring Boot that provides production-ready features. It exposes a series of HTTP endpoints (or JMX beans) that allow you to monitor and manage your application, check its health, view metrics, and understand its internal state without having to write any code.

### Q2. How do you enable Spring Boot Actuator? [Easy]
**Answer:** By adding the `spring-boot-starter-actuator` dependency to your `pom.xml` or `build.gradle`.

### Q3. By default, which Actuator endpoints are exposed over HTTP? [Easy]
**Answer:** For security reasons, only `/health` and `/info` are exposed over HTTP by default.

### Q4. How do you expose all Actuator endpoints over HTTP? [Easy]
**Answer:** You must configure it in your `application.properties` or `application.yml` using the `management.endpoints.web.exposure.include` property.
**Code Example:**
```properties
management.endpoints.web.exposure.include=*
```

### Q5. What does the `/actuator/health` endpoint do? [Easy]
**Answer:** It returns basic application health information. By default, it just returns `{"status": "UP"}`. However, it internally runs various `HealthIndicator`s (e.g., checking if the database connection is alive, or if Redis is reachable). If any critical indicator fails, it returns `{"status": "DOWN"}` and an HTTP 503 status code.

### Q6. What does the `/actuator/info` endpoint do? [Easy]
**Answer:** It displays arbitrary application information. It is empty by default, but you can configure it to display the application version, build time, Git commit hash, or custom static properties defined in your configuration file.

### Q7. What does the `/actuator/metrics` endpoint do? [Easy]
**Answer:** It exposes a massive list of metrics tracked by the application, such as JVM memory usage, CPU usage, Tomcat active sessions, HTTP request execution times, and HikariCP connection pool stats.

### Q8. What does the `/actuator/env` endpoint do? [Easy]
**Answer:** It returns all properties from the Spring `Environment`. This includes system environment variables, JVM properties, and values loaded from `application.properties`/`yml`. It is incredibly useful for debugging configuration issues (e.g., checking which database URL the app actually loaded).

### Q9. What does the `/actuator/beans` endpoint do? [Easy]
**Answer:** It displays a complete list of all Spring Beans configured in the `ApplicationContext`, their scope (singleton, prototype), their dependencies, and the class that defined them.

### Q10. What does the `/actuator/mappings` endpoint do? [Easy]
**Answer:** It provides a collated list of all `@RequestMapping` paths mapped by Spring MVC, showing exactly which controller methods handle which URLs and HTTP methods.

## Intermediate Level Questions

### Q11. How do you secure Actuator endpoints? [Medium]
**Answer:** Because endpoints like `/env` and `/heapdump` expose highly sensitive data, they must be secured. You use **Spring Security**. You define a `SecurityFilterChain` that requires authentication (and specific roles, like `ADMIN`) for any URL matching `/actuator/**`, while leaving your public APIs open.

### Q12. How do you show detailed information (like database connection status) in the `/health` endpoint? [Medium]
**Answer:** By default, details are hidden from anonymous users. You must set `management.endpoint.health.show-details=always` (or `when-authorized`) in your properties file.

### Q13. How do you create a custom `HealthIndicator`? [Medium]
**Answer:** You create a Spring Bean that implements the `HealthIndicator` interface and overrides the `health()` method to return `Health.up().build()` or `Health.down().withDetail("error", "API is unreachable").build()`.
**Code Example:**
```java
@Component
public class ExternalApiHealthIndicator implements HealthIndicator {
    @Override
    public Health health() {
        if (checkApiStatus()) return Health.up().build();
        return Health.down().withDetail("reason", "External API Timeout").build();
    }
}
```

### Q14. What is Micrometer? [Medium]
**Answer:** Micrometer is an open-source metrics facade (similar to what SLF4J is for logging). Spring Boot Actuator uses Micrometer internally to record metrics. Micrometer abstracts away the specific monitoring system, allowing you to seamlessly export metrics to Prometheus, Datadog, New Relic, or CloudWatch just by adding a dependency.

### Q15. How do you integrate Actuator with Prometheus? [Medium]
**Answer:** 
1. Add the `micrometer-registry-prometheus` dependency.
2. Expose the endpoint: `management.endpoints.web.exposure.include=prometheus`.
This enables the `/actuator/prometheus` endpoint, which formats all Micrometer metrics into the specific text-based format that the Prometheus server expects to scrape.

### Q16. Explain the difference between Liveness and Readiness probes in Kubernetes/Spring Boot. [Medium]
**Answer:** 
* **Liveness (`/actuator/health/liveness`)**: Indicates if the internal state is broken (e.g., deadlocked threads, OutOfMemory). If it fails, Kubernetes *restarts* the container.
* **Readiness (`/actuator/health/readiness`)**: Indicates if the application is ready to accept HTTP traffic (e.g., waiting for the database cache to warm up on startup). If it fails, Kubernetes stops routing traffic to it, but does *not* restart it.

### Q17. How do you enable Liveness and Readiness groups in Spring Boot Actuator? [Medium]
**Answer:** If Spring Boot detects it is running in a Kubernetes environment, it enables them automatically. For local testing, you can force them on by setting `management.endpoint.health.probes.enabled=true`.

### Q18. What does the `/actuator/loggers` endpoint do? [Medium]
**Answer:** It allows you to view the configured logging levels (TRACE, DEBUG, INFO, WARN, ERROR) of your application packages. Crucially, it accepts POST requests, allowing you to **dynamically change the logging level at runtime** without restarting the application (e.g., temporarily turning on DEBUG logs for a specific package to troubleshoot a live issue).

### Q19. What does the `/actuator/threaddump` endpoint do? [Medium]
**Answer:** It performs a thread dump, returning the current state (RUNNABLE, WAITING, BLOCKED) and stack trace of every thread running in the JVM. It is invaluable for diagnosing deadlocks or performance bottlenecks.

### Q20. What is a Custom Actuator Endpoint, and how do you create one? [Medium]
**Answer:** You can create your own `/actuator/mycustom` endpoint by creating a Bean annotated with `@Endpoint(id = "mycustom")`. Inside it, you annotate methods with `@ReadOperation` (for GET requests), `@WriteOperation` (for POST requests), or `@DeleteOperation` (for DELETE requests).

## Advanced Level Questions

### Q21. Explain the different Micrometer Meter types (Counter, Gauge, Timer). [Hard]
**Answer:**
* **Counter**: Reports a single value that can only increase (e.g., total number of orders placed, total HTTP 500 errors).
* **Gauge**: Reports the current value of a state that can go up and down (e.g., current memory usage, current active DB connections, current items in a queue).
* **Timer**: Measures both the rate of events and the distribution of their durations (e.g., the execution time of a specific HTTP endpoint, tracking the 95th and 99th percentiles).

### Q22. How do you create a custom Counter metric in your code? [Hard]
**Answer:** You inject the `MeterRegistry` bean into your service. Then, you create/retrieve a counter and increment it.
**Code Example:**
```java
@Autowired MeterRegistry registry;
public void processOrder() {
    registry.counter("orders.processed", "type", "electronics").increment();
}
```

### Q23. What is the `@Timed` annotation? [Hard]
**Answer:** It is a Micrometer annotation. When placed on a method (or a Spring MVC Controller), it automatically wraps the method execution in a `Timer`. It records how many times the method was called, the total time spent, and calculates maximum and average latencies.

### Q24. How do you prevent sensitive properties (like passwords or API keys) from being exposed in `/actuator/env`? [Hard]
**Answer:** Spring Boot automatically masks properties containing words like `password`, `secret`, `key`, and `token` with `******`. You can add custom keys to be masked using the `management.endpoint.env.keys-to-sanitize` property. In Spring Boot 3+, all values are masked by default unless explicitly configured otherwise.

### Q25. What does `/actuator/heapdump` do, and what are the security implications? [Hard]
**Answer:** It generates and downloads a `.hprof` file containing a dump of the entire JVM heap memory. This allows you to analyze memory leaks using tools like Eclipse MAT. *Security Implication*: The heap contains all objects currently in memory, including plain-text passwords, JWT tokens, and PII (Personal Identifiable Information). Exposing this endpoint without strict security is catastrophic.

### Q26. How do you use Actuator to forcefully shut down a Spring Boot application? [Hard]
**Answer:** You can enable the `/actuator/shutdown` endpoint (it is disabled by default). Sending an HTTP POST request to this endpoint triggers a graceful shutdown of the Spring `ApplicationContext` and the embedded web server.
`management.endpoint.shutdown.enabled=true`

### Q27. Explain Spring Boot Admin. [Hard]
**Answer:** Spring Boot Admin (SBA) is an open-source community project (not officially maintained by Pivotal/Spring, but built by Codecentric) that provides a beautiful, centralized UI dashboard. You deploy SBA as a standalone server. Your other Spring Boot microservices register with it as clients. SBA reads all their Actuator endpoints and displays them visually (graphs, tables, log viewers, thread dump visualizers).

### Q28. How does Actuator integrate with JMX (Java Management Extensions)? [Hard]
**Answer:** Before HTTP APIs became standard, JMX was the primary way to monitor Java applications. Actuator exposes all its endpoints as JMX MBeans by default. You can connect to the running application using `jconsole` or `VisualVM` and interact with the endpoints (like viewing metrics or triggering a heap dump) via the JMX protocol instead of HTTP.

### Q29. How do you optimize Actuator endpoints to reduce performance overhead? [Hard]
**Answer:** Actuator polling (especially by Prometheus every 5 seconds) can add CPU/Memory overhead.
1. Use `management.endpoints.web.exposure.include` to expose *only* the endpoints you actively use (e.g., just `health` and `prometheus`).
2. Cache the health indicator results using `management.endpoint.health.cache.time-to-live=10s` so multiple rapid requests don't trigger heavy database pings.
3. Turn off HTTP request tracing (`/httptrace`) if you are using a dedicated tool like Micrometer Tracing.

### Q30. How do you implement a push-based metrics export instead of a pull-based one? [Hard]
**Answer:** Prometheus is *pull-based* (it scrapes `/actuator/prometheus`). However, systems like Datadog, New Relic, or InfluxDB are *push-based*. Micrometer handles this automatically. If you add the `micrometer-registry-datadog` dependency and provide an API key in properties, Micrometer starts a background thread that periodically (e.g., every 1 minute) batches the metrics and POSTs them to the Datadog API servers.
